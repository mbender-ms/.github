# Gold-standard PR example

---

## Docs PR format

```markdown
## Summary

{1-2 sentence overview of what changed and why.}

## Staged articles

1. [Article H1 Title](https://review.learn.microsoft.com/en-us/{path}?view={view}&branch=pr-en-us-{PR}#{bookmark})
2. [Another Article Title](https://review.learn.microsoft.com/en-us/{path}?view={view}&branch=pr-en-us-{PR}#{bookmark})

## Changes

### Commit 1: {commit message} ({file count} files, +{additions}/-{deletions})

**1. {folder/article-file.md}**
- Added [{Heading text}]({staging-url}#{bookmark}) section with {brief description}
- Updated [{Heading text}]({staging-url}#{bookmark}) with {what changed}

### Commit 2: Edits based on feedback ({file count} files, +{additions}/-{deletions})

**1. {folder/article-file.md}**
- Updated [{Heading text}]({staging-url}#{bookmark}): "{old wording}" → "{new wording}"

---

## Related work items

AB#{work-item-id}

> [!NOTE]
> This PR description was generated by CDA Cortex and may require review for accuracy.
```

## Code PR format

```markdown
## Summary

{What changed and why — reference the problem being solved.}

## Changes

### Commit 1: {commit message} ({file count} files, +{additions}/-{deletions})

**1. {src/services/pr-generator.ts}**
- Removed Impact and Files Modified sections (redundant with GitHub's Files tab)
- Made Testing section conditional on PR type (code/mixed only)

**2. {src/tools/handlers/generate-pr-description.ts}**
- Updated template selection to pass PR type through to body builder

### Commit 2: {commit message} ({file count} files)

**1. {src/__tests__/pr-generator.test.ts}**
- Added tests for docs-only PR (no Testing section) and code PR (with Testing)

---

## Testing

- All 48 test suites pass (859 tests)
- Manually verified: docs PR produces no Testing section, code PR includes checklist

## Related work items

AB#{work-item-id}

> [!NOTE]
> This PR description was generated by CDA Cortex and may require review for accuracy.
```

## Key patterns

1. **Choose sections by PR type** — docs PRs get staged articles but no Testing; code PRs get Testing but no staged articles; mixed PRs get both
2. **Per-commit sections** — `### Commit N: {message} ({stats})`. Group feedback rounds logically (can label "Edits based on feedback")
3. **Per-file entries** — `**N. {path}**` followed by bullet points describing what changed and why
4. **Staging links** (docs only) — every bullet has an inline link `[Heading text](url#bookmark)`. Validate bookmarks against actual file headings
5. **`?view=` param** (docs only) — determined by docset (`azuresql`, `sql-server-ver16`, `azure-devops-rest-7.1`, etc.)
6. **Preserve standard sections** — Summary, Related work items, and RAI notice from the baseline PR must be preserved verbatim
7. **Section order** — Docs: Summary → Staged articles → Changes → Related work items → RAI. Code: Summary → Changes → Testing → Related work items → RAI
