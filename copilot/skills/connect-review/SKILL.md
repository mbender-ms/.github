## Overview

This skill generates a polished HTML Connect document — the biannual performance review artifact submitted through Microsoft's Connects tool. The document ties the user's individual SMART goals and accomplishments to the organizational goal hierarchy:

```
Azure Core (Girish) > AX&E (Annie) > Learn (Hans) > Learn Content (Martin) > EPD/ADEC (Josée) > M1 teams
```

The Connect is NOT a metrics dump. It's a narrative document that demonstrates:
1. **What you committed to** (SMART goals)
2. **What you delivered** (evidence with metrics)
3. **Why it matters** (business impact tied to org goals)
4. **What level you operated at** (IC level callouts)

## Directory Structure

```
Reports/Connects/{Connect Name}/
├── reference-files/                    ← Flat folder for user-provided docs + GOALS.md
│   ├── GOALS.md                        ← Copied from connect-goals skill
│   ├── *-content-goals.md              ← Org content goals (user provides)
│   └── *-north-star.md                 ← Org north star (user provides)
└── output/
    ├── connect-metrics/                ← Intermediate JSON data files (auto-generated by connect-metrics skill)
    ├── connect-metrics-data.html       ← Metrics dashboard HTML (auto-generated by connect-metrics skill)
    ├── connect-output.html             ← Full comprehensive HTML with IC callouts (generated by this skill)
    └── connect-text-for-tool.html      ← Paste-friendly version for Connects tool (generated by this skill)
```

**SMART goals source:** The `connect-goals` extension skill generates and maintains `GOALS.md` (`~/.copilot/skills/connect-goals/GOALS.md`) with auto-refreshed progress counters and dated evidence entries. If the user has been tracking goals with CDA throughout the semester, this file is the primary source and just needs to be copied over. If not, the user can provide their goals manually.

**Metrics data:** The `connect-metrics` skill auto-gathers all metrics including Power BI data, ADO work items, and GitHub PRs. The user does not need to provide this data.

**Sample files for reference**: `Reports/Connects/Sample/` contains:
- `SAMPLE-connect-text-for-tool.html` — Complete example paste version (fictional data)
- `SAMPLE-connect-review-data.html` — Example data report

## Step 1: Understand the Goal Hierarchy

Read the reference files and build a mental model of how goals cascade:

### The 3 Uber Goals (from Content Goals doc)
These are the TOP-LEVEL sections of the Connect. Every SMART goal must map to one:

1. **DELIVER SECURE, HIGH-QUALITY CONTENT FOR CUSTOMERS** — Core content work, feature docs, UUF, quality
2. **CONTINUOUSLY IMPROVE AND INNOVATE** — AI adoption, tooling, process improvement, UATs
3. **WORK BETTER TOGETHER WITH PEERS & PARTNERS** — Team resources, community events, collaboration

### Org Alignment Chain
For each accomplishment, trace the value UP the chain:
- **Your work** → supports **EPD/ADEC key result** → supports **Learn Content investment area** → supports **Learn strategic choice** → supports **AX&E priority** → supports **Azure Core priority**

Example:
- "Documented {Area1} feature X GA" → EPD KR: committed content on time → Learn Content: shipping high-quality content → Learn: paths to value → AX&E: accelerate product adoption → Azure Core: Growth

### North Star Alignment
Each goal section should touch on relevant North Star focus areas:
- **Customers** — listening to feedback, UUF resolution
- **Content** — lifecycle management, quality, retirement
- **AI** — AI tools in workflow, agentic workflows
- **Innovation** — experimentation, new tools, UATs
- **Team** — mentoring, knowledge sharing, collaboration

## Step 2: Map SMART Goals to Sections

Read the user's SMART goals file. For each goal:

1. **Identify which uber goal it belongs to** (Goal 1, 2, or 3)
2. **Read the self-assessment** — determine outcome: Met, Exceeded, Partially Met, Not Met (with explanation)
3. **Find supporting metrics** from connect-review-data:
   - ADO work items related to this goal
   - PRs merged related to this goal
   - Lines added/removed, articles created/updated
   - UUF items closed (for UUF goals)
4. **Find narrative context** from session summaries and monthly reports:
   - Key decisions made
   - Blockers encountered and resolved
   - Cross-team collaboration details
   - Customer impact stories
5. **Identify org alignment** — which EPD key results and Content North Star areas does this goal support?
6. **Determine IC level demonstration** — what career framework responsibility does this work demonstrate?

Present the mapping to the user for review before proceeding to HTML generation.

## Step 3: Generate the Connect HTML

### Output Location
Save to: `{Connect folder}/connect-output.html`

### HTML/CSS Foundation
Use `SAMPLE-connect-text-for-tool.html` from the Sample folder (or user's own `*-template-reference.html` if they have one) as the structural reference:
- Same purple gradient body background
- Same white container with border-radius 16px
- Same metric cards, scorecard grid, bar charts, impact cards
- Same dark mode toggle, TOC sidebar, back-to-top button, expand/collapse

### Teams/OneDrive Compatibility

HTML files opened in Teams, OneDrive, or SharePoint web viewers have JavaScript blocked or sandboxed. The HTML needs to work fully without JS — otherwise metrics show "0" and sections are invisible when managers open the file from a Teams link.

**Rules for JS-free compatibility:**

1. **Metric counters**: Show the final number directly in the HTML text content — don't rely on `data-target` + JS animation to display values. Use `data-target` as an enhancement only; the visible text should already be the correct number:
   ```html
   <!-- CORRECT: Number visible without JS -->
   <div class="metric-value" data-target="85">85</div>
   
   <!-- WRONG: Shows "0" without JS -->
   <div class="metric-value" data-target="85">0</div>
   ```

2. **Section reveal animations**: Add `.revealed` class by default via CSS, not JS. The `IntersectionObserver` reveal is a progressive enhancement only:
   ```css
   /* Sections visible by default — JS adds animation if available */
   .section { opacity: 1; transform: none; }
   ```
   If you want the animation for browser viewing, add it via JS by first setting opacity:0 and then observing, rather than starting hidden in CSS.

3. **`<details>` elements**: These work natively in HTML without JS — collapsible sections are fine.

4. **Dark mode toggle**: Keep the button but ensure the default light theme renders correctly without JS. `localStorage` theme persistence is a progressive enhancement.

5. **TOC sidebar active state**: The sidebar links work as anchor navigation without JS. Active-state highlighting is a progressive enhancement only.

6. **Back-to-top button**: Use `<a href="#top">` as fallback in addition to the JS `scrollTo`. Hide via CSS initially, show with JS.

7. **`onclick` handlers**: These may be stripped in sandboxed viewers. Keep all content visible without any click interaction — toggle features (dark mode, expand/collapse) are optional enhancements.

**Testing**: Open the generated HTML in a OneDrive/Teams preview to verify content renders correctly before finalizing.

### Section-by-Section Guide

---

#### HEADER
**Purpose:** Identify the author, period, role, and manager at a glance.

**Contains:**
- Title: "📋 Connect {Month} {Year}"
- Subtitle: Team name (e.g., "{Team Name} | {Team Org}")
- Meta items: Author name + email, period dates, role title, manager name

**References:** Agent file (name, email, role, manager)

---

#### SECTION 1: Period Summary
**Purpose:** Executive summary — 2-3 sentences capturing the headlines of the semester. This is what the skip-level reads first.

**Contains:**
- Highlight box with bold key stats
- Total work items closed (from connect-review-data)
- Number of SMART goals achieved / exceeded / partially met
- 1-2 standout accomplishments that demonstrate impact beyond routine work
- Conference deliverables if applicable

**References:** Connect-review-data metrics, SMART goals self-assessment, monthly reports for narrative

**Writing guidance:**
- Lead with outcomes, not activities
- Use specific numbers
- Mention goals exceeded (these catch manager attention)
- Keep to 3-4 sentences max

---

#### SECTION 2: SMART Goals Scorecard
**Purpose:** At-a-glance status of every SMART goal — lets manager quickly see green/exceeded/tracking.

**Contains:**
- Scorecard grid with one card per SMART goal
- Each card shows: goal label, brief status line, badge (✅ Done, 🌟 Exceeded, 🔄 On Track, ⚠️ Partially Met, ❌ Not Met)
- Color coding: green = done, blue = exceeded, yellow = on track, orange = partial, red = not met

**References:** SMART goals file (all goals + self-assessment)

**Scoring guidance:**
- **Exceeded (🌟)**: Quantifiably surpassed the target (e.g., goal was 35% UUF reduction, achieved 39%)
- **Done (✅)**: Met the goal as stated
- **On Track (🔄)**: Ongoing commitment being maintained (e.g., monthly contributions)
- **Partially Met (⚠️)**: Made progress but didn't fully achieve — explain what was accomplished and what blocked completion
- **Not Met (❌)**: Did not achieve — explain external factors (PM delays, org changes, etc.)

---

#### SECTION 3: Cumulative Metrics
**Purpose:** Quick visual of key numbers for the semester.

**Contains:**
- Metric cards grid (6-8 cards): Items Closed, UUF Resolved, PRs Authored, PRs Reviewed, New Articles, Articles Updated
- Use animated counters (data-target attribute) but show real values in HTML text (Teams compatibility)
- Color-code cards: primary, accent, success, warning

**References:** Connect-review-data (`connect-review-data.html` key metrics section)

---

#### SECTION 3b: Combined Metrics Table (Paste Version Only)
**Purpose:** Detailed per-product-area breakdown for the Connects tool paste version. Replaces the simple metrics cards.

**Copy the Combined Metrics Table HTML directly from `connect-review-data/connect-review-data.html`** rather than recreating it manually — the connect-metrics skill generates this table with verified, cross-referenced data, so copy-paste ensures consistency.

**Contains:**
- Full combined metrics table from connect-review-data.html (products as columns, metrics as rows)
- Include: Avg monthly visitors, Avg monthly page views, Articles owned, ADO items, Items closed, UUF resolved, User Stories closed, PRs, New articles, Updated articles, Lines added/removed, Images, Conference PRs, New features, Features going GA, Features in preview
- Skip redundant rows (Feature PRs, Maintenance PRs, Maintenance items)
- Use `<sup>` footnotes for co-owned content and total calculations
- Use simple `<table border="1">` for paste compatibility (no CSS classes)
- Reduce padding from the source table: change `cellpadding="6"` to `cellpadding="4"` and remove `width:100%`

**References:** Connect-review-data combined metrics table — copy directly rather than recreating

---

#### SECTION 4: Semester Work Distribution
**Purpose:** Show how time/effort was allocated across work categories.

**Contains:**
- Horizontal bar chart with percentage fills
- Categories derived from product areas + special projects
- Each bar has gradient fill, label, and percentage

**References:** Connect-review-data (PR counts by category, ADO items by area)

**Guidance:** Categories should reflect meaningful work streams, not just product areas. Group small areas together. Include special projects (e.g., "UUF Resolution", "PR Reviews", "Conference Docs") alongside product areas.

---

#### SECTION 5: Goal Detail Sections (THE CORE)
**Purpose:** This is the meat of the Connect. One major section per uber goal, containing detailed write-ups of each SMART goal with evidence, impact, and IC-level callouts.

**Goal 1 Structure (unique — differs from Goals 2 and 3):**
Goal 1 has a special structure that differs from Goals 2 and 3:

1. **Product Area Summaries** (BEFORE individual SMART goals)
   - One paragraph per product area ({Area1}, {Area2}, {Area3}, {Area4})
   - Combine shared content areas where applicable (e.g., if two ms.service values share an area)
   - Each summary: work item/PR count, PG alignment, high-level impact
   - Followed by `Notable features:` and `Notable maintenance:` bullet lists
   - Pull from connect-review-data executive summaries
   - Ensure correct product area attribution (see connect-review.SKILL.md Phase 10 rules)

2. **Conference Contributions** (AFTER product summaries, BEFORE SMART goals)
   - Table: Conference, PRs, New, Updated, +Lines, -Lines, Images
   - Notable features per conference as text below the table
   - Pull from connect-review-data conference table

3. **Individual SMART goals** (1.1, 1.2, 1.3, etc.)
   - Standard goal-section format (see below)

**Goals 2 and 3 Structure:**
Standard SMART goal sections, PLUS **unplanned goals** for significant work not in original SMART goals:

- **Unplanned Goal 2 example**: Partner Publishing Workflow (process innovation)
  - Include metrics (PRs caught, auto-merge prevention rate)
  - Include peer quotes woven into narrative (not in a separate section)
  - Include cross-team adoption evidence

- **Unplanned Goal 3 examples**: Link Farm Elimination (team project contribution), PR Reviews & Peer Coverage
  - For team projects: note it was a team effort, cite your specific contribution vs. team total
  - Include peer/manager quotes naturally in context

**Peer Quotes — weave into goal sections, don't list separately:**
A standalone "Peer Feedback" section reads like padding. Instead, weave quotes into the relevant goal sections where the work happened:
- Workflow feedback → into the workflow section
- Manager praise → into team resources or period summary
- Vendor/editor thanks → into PR reviews section
- Cross-team recognition → into the section where that work is described

**Structure per uber goal:**

```
<details open class="section" id="goal{N}">
  <summary><h2>🎯 Goal {N}: {Uber Goal Title}</h2></summary>

  <!-- One goal-section div per SMART goal -->
  <div class="goal-section">
    <h3>{N}.{M} {Icon} {SMART Goal Title}</h3>
    <div class="badges">ADO links, PR links, status badge</div>
    <p>Narrative paragraph(s) describing what was done and why</p>

    <!-- Evidence table or deliverables list -->
    <table>...</table>  OR  <ul class="deliverables">...</ul>

    <!-- Why It Matters box -->
    <div class="why-it-matters">
      <h4>💡 Why It Matters</h4>
      <p>Connect this work to business impact, customer value, org priorities</p>
    </div>

    <!-- IC Level callout box -->
    <div class="ic-level-box">
      <h4>🎯 IC Level: IC{N} — {Responsibility Area}</h4>
      <p>Quote from career framework showing how this work demonstrates that level</p>
    </div>
  </div>
</details>
```

**For each SMART goal write-up:**

1. **Title line**: Include icon, goal name, and exceeded badge if applicable
2. **Badges**: Link to ADO work items and key PRs (clickable links, not just numbers)
3. **Narrative**: 1-2 paragraphs written in third person describing:
   - What was committed to
   - What was actually delivered (with specific numbers)
   - How the work was approached (methodology, tools, collaboration)
   - Notable challenges overcome
4. **Evidence**: Table or bullet list with specific deliverables:
   - For feature work: table with Feature, Conference, ADO#, PR#, Impact columns
   - For UUF: table with Month, Items, Key PRs, Coverage columns
   - For team contributions: chronological bullet list of specific actions with dates
5. **Why It Matters box**: 2-3 sentences connecting this work to:
   - Business impact (revenue, adoption, customer success)
   - Org priority (EPD key result, North Star area, AX&E investment)
   - Customer impact (reduced friction, improved accuracy, security)
6. **IC Level callout**: 
   - Identify which IC4/IC5 responsibility area this work best demonstrates
   - Quote the specific language from the career framework (from promo/references/)
   - Explain how the work maps to that language
   - For work that stretches into IC5 territory, use "IC4 → IC5" notation

**References for each goal:**
- SMART goals file (commitment + self-assessment)
- Connect-review-data (ADO items, PRs, metrics for that area)
- Session summaries (narrative context, decisions, blockers)
- Monthly reports (big rocks, accomplishments)
- EPD plan (which key results this supports)
- North Star (which focus areas this touches)
- Career framework files in promo/references/ (IC level quotes)

**Writing guidance for goal sections:**
- **DO**: Lead with impact, use specific numbers, cite PG goals, reference conferences
- **DO**: Show methodology and approach, not just output
- **DO**: Acknowledge blockers honestly (especially for "Not Met" goals — show what you DID do)
- **DON'T**: Just list tasks — frame everything in terms of value delivered
- **DON'T**: Overclaim — be precise about your specific contribution vs. team effort
- **DON'T**: Skip the "Why It Matters" — this is what differentiates a good Connect from a great one

**Handling "Not Met" or "Partially Met" goals:**
- Be honest and direct about what happened
- Explain external factors (PM turnover, org changes, blocked by PG)
- Emphasize what WAS accomplished despite blockers
- Show proactive response (e.g., "wrote one feature article for the first step")
- Frame as learning opportunity or readiness for when blocker clears
